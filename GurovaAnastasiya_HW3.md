# Краткое руководство пользователя git

## Первый запуск
**git init** - команда, запускающая git в соответствующей папке

**git add** *filename.ext* - команда, добавляющая соответствующий файл к отслеживанию 

## Фиксация и просмотр изменений
! После каждого существенного изменения и сохранения файла необходимо "обновить" его в репозитории путем запуска команды **git add** и добавления фиксации командой **git commit**.

**git commit -m "комментарий"** - команда, добавляющая фиксацию текущего состояния (сохранение) с обязательным вводом комментария

**git log** - команда вызова журнала изменений позволяет просмотреть сделанные сохранения файла

**git status** - команда, дающая сведения о папке, в которой ведется контроль версий

## Переход к сохраненной версии
**git checkout xxxxx** - команда, позволяющая перейти к нужной сохраненной версии (выбирается из журнала, полученного запуском **git log**, по первым 5 и более символам в номере)

**git checkout master** - команда, возвращающая к актуальной версии

## Анализ различий между версиями
**git diff** - команда, показывающая разницу между сохраненной и текущей версиями файла

## Ветвление
### 1. Создание новых веток и переход между ними
**git branch** - команда для просмотра существующих веток

**git branch** *branchname* - команда для создания новой ветки с названием branchname

**git branch -d** _branchname_ - команда для удаления ветки с названием *branchname*

### 2. Слияние веток
! Для начала слияния следует сначала перейти в нужную ветку.

**git merge** *branchname* - команда, добавляющая в текущую ветку инфо из ветки с названием branchname
### 3. Конфликты при слиянии веток
* Конфликты при слиянии возникают, когда коммиты сливаемых веток содержат противоречие 
* Можно добавить в репозиторий файл .gitignore, содержащий список файлов, которые не нужно коммитить, например графические файлы.  
* После добавления файла .gitignore его нужно закоммитить (__git add__ + __git commit__) 

## Работа с удаленными репозиториями
**GitHub** - сервис Microsoft, который позволяет интегрироваться с git и удаленно хранить и работать с репозиториями.
### 1. Создание удаленного репозитория через Visual Studo Code
! Сначала нужно зарегистрироваться на Github.com и создать там новый репозиторий.
1. Создаем локальный репозиторий в новой папке через Visial Studio Code (**git init** + **git add** + **git commit**).
2. Команды для установки связи между удаленным и локальным репозиторием:

**git remote add origin** *ссылка на репозиторий в GitHub* - указание адреса репозитория в GitHub

**git branch -M master** - команда, определяющая основную ветку

**git push -u origin main** - команда, отправляющая данные в репозиторий

### 2. Работа с "чужим" репозиторием
1. Делаем **fork** нужного репозитория на GitHub.
2. Делаем **clone** своей версии репозитория удобным способом в Visual Studio Code.
3. Создаем НОВУЮ ветку и в нее вносим изменения (**git branch** *НОВАЯ ветка*).
4. Фиксируем изменения (**git add** + **git commit**).
5. Отправляем свою версию в GitHub (**git push**).
6. На GitHub отправляем **Pull request**.
